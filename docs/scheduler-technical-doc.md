## 二、任务模型定义

### 2.1 核心数据结构

#### 任务模式 (TaskMode)

```typescript
// 两种任务模式
type TaskMode = 'simple' | 'protocol';

// simple: 简单模式 - 直接使用 prompt 字段作为提示词
// protocol: 协议模式 - 读取 task.md + memory + supplement 构建提示词
```

#### 触发类型 (TriggerType)

```typescript
type TriggerType = 'once' | 'cron' | 'interval';

// once: 单次执行 - ISO 时间戳 (如 "2024-03-16T14:00:00Z")
// cron: Cron 表达式 (如 "0 9 * * 1-5" 表示工作日早9点)
// interval: 间隔执行 (如 "30s", "5m", "2h", "1d")
```

#### ScheduledTask 完整结构

```typescript
interface ScheduledTask {
  id: string;                          // 任务唯一ID
  name: string;                        // 任务名称
  enabled: boolean;                    // 是否启用
  triggerType: TriggerType;            // 触发类型
  triggerValue: string;                // 触发值
  engineId: string;                    // 使用的 AI 引擎ID

  // 模式相关
  mode: TaskMode;                      // 任务模式
  prompt: string;                      // 简单模式的提示词
  workDir?: string;                    // 工作目录
  taskPath?: string;                   // 协议模式任务路径
  mission?: string;                    // 任务目标 (协议模式)

  // 执行状态
  lastRunAt?: number;                  // 上次执行时间戳
  lastRunStatus?: TaskStatus;          // 上次执行状态
  nextRunAt?: number;                  // 下次执行时间戳
  currentRuns: number;                 // 已执行轮次
  retryCount: number;                  // 已重试次数

  // 执行控制
  maxRuns?: number;                    // 最大执行轮次
  maxRetries?: number;                 // 最大重试次数
  retryInterval?: string;              // 重试间隔
  timeoutMinutes?: number;             // 超时时间(分钟)
  runInTerminal: boolean;              // 是否在终端执行

  // 订阅功能
  subscribedContextId?: string;        // 订阅的上下文ID

  // 通知
  notifyOnComplete?: boolean;          // 完成后是否通知

  // 用户补充
  userSupplement?: string;             // 一次性提示词
}
```

## 四、协议模式定时任务详解

### 4.1 协议模式概述

协议模式是定时任务系统的高级功能，用于持续性的复杂任务。它通过文件系统维护任务状态，支持：
- 任务目标追踪
- 执行记忆持久化
- 用户临时干预
- 文档自动备份

### 4.2 协议任务目录结构

```
{workDir}/
├── .polaris/
│   └── tasks/
│       └── {timestamp}/              # 如 20240316143000
│           ├── task.md               # 协议文档（任务目标、规则）
│           ├── user-supplement.md    # 用户补充内容
│           └── memory/
│               ├── index.md          # 成果索引（已完成、进行中）
│               └── tasks.md          # 任务队列（待办、已完成）
│
└── .oprcli/
    └── tasks/
        └── {timestamp}/
            ├── supplement-history/   # 用户补充历史备份
            │   └── 20240316-150000.md
            └── doc-history/          # 文档历史备份
                └── task-20240316-160000.md
```

```
每次触发时按以下顺序执行：

### 1. 检查用户补充
- 读取 `.polaris/tasks/{timestamp}/user-supplement.md`

### 2. 推进主任务

- 完成具体功能实现
- 修复已知问题
- 优化代码质量
- 产出可复用资产

- 无产出的探索
- 重复性工作
```

## 五、触发机制详解

### 5.1 触发类型与计算

```rust
impl TriggerType {
    pub fn calculate_next_run(&self, trigger_value: &str, now: i64) -> Option<i64> {
        match self {
            // 单次执行：解析 ISO 时间戳
            TriggerType::Once => {
                chrono::DateTime::parse_from_rfc3339(trigger_value)
                    .ok()
                    .map(|dt| dt.timestamp())
                    .filter(|&ts| ts > now) // 只返回未来的时间
            }

            // Cron 表达式：使用 cron 库计算
            TriggerType::Cron => {
                Schedule::from_str(trigger_value)
                    .ok()
                    .and_then(|schedule| {
                        schedule.upcoming(chrono::Utc).next()
                            .map(|dt| dt.timestamp())
                    })
            }

            // 间隔执行：解析间隔表达式
            TriggerType::Interval => {
                parse_interval(trigger_value)
                    .map(|interval_secs| now + interval_secs)
            }
        }
    }
}

// 间隔解析：支持 "30s", "5m", "2h", "1d"
pub fn parse_interval(value: &str) -> Option<i64> {
    let (num_str, unit) = value.split_at(value.len() - 1);
    let num: i64 = num_str.parse().ok()?;

    let multiplier = match unit.to_lowercase().as_str() {
        "s" => 1,           // 秒
        "m" => 60,          // 分钟
        "h" => 3600,        // 小时
        "d" => 86400,       // 天
        _ => return None,
    };

    Some(num * multiplier)
}
```

### 5.2 下次执行时间更新

```
任务执行完成后更新 nextRunAt:
┌─────────────────────────────────────────────────────────────────┐
│  if status == Success {                                         │
│    currentRuns++                                                │
│    if currentRuns >= maxRuns {                                  │
│      enabled = false  // 达到最大轮次，自动禁用                  │
│      nextRunAt = None                                           │
│    } else {                                                     │
│      nextRunAt = calculate_next_run(trigger_type, value, now)   │
│    }                                                            │
│  } else if status == Failed {                                   │
│    if maxRetries > 0 && retryCount < maxRetries {               │
│      retryCount++                                               │
│      nextRunAt = now + parse_interval(retryInterval)            │
│    } else {                                                     │
│      nextRunAt = calculate_next_run(...)                        │
│    }                                                            │
│  }                                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 前端事件监听

```typescript
// schedulerStore.ts
initSchedulerEventListener: (getCurrentContextId) => {
  const handleSchedulerEvent = (event) => {
    const { type, taskId, taskName, success, contextId } = event.payload;

    if (type === 'task_end') {
      // 任务结束，清除订阅状态
      set({ subscribingTaskId: null, subscribingTaskName: null });

      // 刷新任务列表和日志
      get().loadTasks();
      get().loadLogs(50);

      // 自动续订：更新 contextId 为当前活动会话
      if (success && getCurrentContextId) {
        const currentContextId = getCurrentContextId();
        if (currentContextId) {
          tauri.schedulerSubscribeTask(taskId, currentContextId);
        }
      }
    } else if (type === 'task_due') {
      // 任务到期，自动执行
      set({ subscribingTaskId: taskId, subscribingTaskName: taskName });
      tauri.schedulerRunTaskWithWindow(taskId, contextId);
    }
  };

  // 监听 scheduler-event 事件
  tauri.listen('scheduler-event', handleSchedulerEvent);
}
```
