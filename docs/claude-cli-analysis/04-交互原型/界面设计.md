# 交互原型设计：Claude CLI 可视化界面（深度修订版）

**版本**: v2.0
**日期**: 2026-04-15
**设计原则**: 数据驱动 + 渐进展示 + 状态可见

---

## 一、全局设计原则

1. **数据驱动**: 所有列表数据从 CLI 动态获取，UI 仅展示数据
2. **渐进展示**: 核心信息优先展示，高级选项折叠
3. **状态可见**: CLI 连接状态、认证状态、MCP 状态实时可见
4. **操作反馈**: 每个 CLI 操作都有明确的 loading/success/error 反馈
5. **来源区分**: 插件管理的 MCP vs 用户手动添加的 MCP 明确区分
6. **一致性**: 与现有 Polaris 设置页风格统一，复用已有组件

---

当前:

## 六、交互流程图

### 6.1 应用启动数据流

```
App 启动
  │
  ├─→ configStore 初始化
  │     └─→ 获取 cliPath
  │
  ├─→ cliInfoStore.fetchAll(cliPath)
  │     ├─→ get_cli_version()     → 缓存版本号
  │     ├─→ get_cli_agents()      → 解析并缓存 Agent 列表
  │     └─→ get_cli_auth_status() → 缓存认证状态
  │
  ├─→ sessionConfigStore.initialize()
  │     └─→ 从 cliInfoStore 获取 Agent/Model 数据
  │         替代硬编码的 PRESET_AGENTS/PRESET_MODELS
  │
  ├─→ pluginStore.fetchInstalled()  (已有)
  │
  └─→ 渲染 UI
        ├─→ AIEngineTab: 显示认证 + Agent 列表 + 版本
        ├─→ ChatStatusBar: 显示动态 Agent/Model 选择器
        └─→ MCPTab: 显示 MCP 服务器列表
```

### 6.2 首次会话 — stream-json init 补充

```
用户发送首条消息
  │
  ├─→ start_chat (已有流程)
  │     └─→ CLI 启动 → stream-json 开始输出
  │
  ├─→ 解析 init 事件 (新增)
  │     ├─→ agents[]     → 更新 cliInfoStore.agents
  │     ├─→ tools[]      → 更新 cliInfoStore.tools
  │     ├─→ mcp_servers[] → 更新 cliInfoStore.mcpServers
  │     ├─→ skills[]     → 更新 cliInfoStore.skills
  │     └─→ model        → 更新 cliInfoStore.currentModel
  │
  └─→ UI 自动更新 (响应式)
        ├─→ 工具权限面板: 显示完整工具列表
        ├─→ MCP Tab: 显示连接状态
        └─→ Agent 选择器: 确认列表完整
```

### 6.3 MCP 服务器添加流程

```
用户点击 [+ 添加 MCP 服务器]
  │
  ├─→ 弹出 AddMCPServerModal
  │     ├─→ 输入名称
  │     ├─→ 选择传输类型 (stdio/http/sse)
  │     ├─→ 根据类型显示配置表单
  │     ├─→ 选择作用域 (user/project/local)
  │     └─→ 底部显示对应的 CLI 命令预览
  │
  ├─→ 点击 [添加]
  │     ├─→ 调用 add_mcp_server()
  │     │     └─→ claude mcp add --transport <type> --scope <scope> \
  │     │           [-e KEY=VALUE...] [-H "Header: Value"] \
  │     │           <name> <commandOrUrl> [-- args...]
  │     ├─→ 成功 → 刷新列表 → toast "MCP 服务器已添加"
  │     └─→ 失败 → 显示错误信息 (stderr 内容)
  │
  └─→ 列表自动刷新
        └─→ list_mcp_servers() → 更新 UI
```

### 6.4 MCP 认证流程

```
用户看到 MCP 服务器状态: "⚠ 需要认证"
  │
  ├─→ 点击 [认证]
  │     ├─→ HTTP 类型: 打开系统浏览器 OAuth 流程
  │     │     └─→ claude mcp get <name> 检查状态
  │     └─→ stdio 类型: 可能需要配置环境变量
  │
  ├─→ 浏览器 OAuth 完成后
  │     └─→ 自动刷新状态 → "✓ Connected"
  │
  └─→ 或手动点击刷新
```

### 6.5 认证管理流程

```
AIEngineTab 显示认证状态
  │
  ├─→ 已登录:
  │     ✅ 已登录 · OAuth 认证 · Anthropic 第一方服务
  │     [登出] → claude auth logout → 刷新状态
  │
  └─→ 未登录:
        ❌ 未登录
        [登录] → 选择方式:
        ├─→ Claude 订阅 (--claudeai)
        ├─→ Anthropic Console (--console)
        └─→ SSO (--sso)
              └─→ 打开浏览器 → 登录完成 → 自动刷新
```
