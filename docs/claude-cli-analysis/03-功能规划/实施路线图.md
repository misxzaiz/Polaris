# 功能规划：Claude CLI 可视化实施路线图（深度修订版）

**版本**: v2.0
**日期**: 2026-04-15
**核心策略**: 利用 stream-json init 事件作为主数据源 + 独立命令作为补充

---

## 一、实施原则

1. **init 事件优先**: stream-json 的 init 事件已包含 agents/tools/mcp_servers/skills/plugins 全部数据，应作为主数据源
2. **渐进增强**: 分阶段实施，每阶段可独立交付
3. **用户无感**: CLI 数据获取在后台完成，不阻塞 UI
4. **降级友好**: CLI 不可用时显示缓存数据 + 提示 + 手动刷新

**新文件**: `src-tauri/src/commands/cli_info.rs`

```rust
use std::process::Command;
use serde::{Deserialize, Serialize};

#[derive(Debug, Serialize, Deserialize)]
pub struct CliAgent {
    pub id: String,
    pub name: String,
    pub source: String, // "builtin" | "plugin"
    pub default_model: Option<String>,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct AuthStatus {
    pub logged_in: bool,
    pub auth_method: String,
    pub api_provider: String,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct CliInitData {
    pub agents: Vec<String>,
    pub tools: Vec<String>,
    pub mcp_servers: Vec<McpServerStatus>,
    pub skills: Vec<String>,
    pub model: String,
    pub version: String,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct McpServerStatus {
    pub name: String,
    pub status: String, // "connected" | "needs-auth" | "error" | "pending"
}

#[tauri::command]
pub async fn get_cli_agents(cli_path: String) -> Result<Vec<CliAgent>, String> {
    let output = Command::new(&cli_path)
        .args(["agents"])
        .output()
        .map_err(|e| format!("CLI 执行失败: {}", e))?;

    let stdout = String::from_utf8_lossy(&output.stdout);
    parse_agents_output(&stdout)
}

#[tauri::command]
pub async fn get_cli_auth_status(cli_path: String) -> Result<AuthStatus, String> {
    let output = Command::new(&cli_path)
        .args(["auth", "status"])
        .output()
        .map_err(|e| format!("CLI 执行失败: {}", e))?;

    serde_json::from_slice(&output.stdout)
        .map_err(|e| format!("解析失败: {}", e))
}

#[tauri::command]
pub async fn get_cli_version(cli_path: String) -> Result<String, String> {
    let output = Command::new(&cli_path)
        .args(["--version"])
        .output()
        .map_err(|e| format!("CLI 执行失败: {}", e))?;

    Ok(String::from_utf8_lossy(&output.stdout).trim().to_string())
}

fn parse_agents_output(output: &str) -> Result<Vec<CliAgent>, String> {
    let mut agents = Vec::new();
    let mut source = "builtin".to_string();

    for line in output.lines() {
        let trimmed = line.trim();
        if trimmed.starts_with("Plugin agents:") {
            source = "plugin".to_string();
            continue;
        }
        if trimmed.starts_with("Built-in agents:") {
            source = "builtin".to_string();
            continue;
        }
        // 匹配: "  agent-name · model"
        if let Some(captures) = regex::Regex::new(r"^(\S+)\s+·\s+(\S+)$")
            .ok()
            .and_then(|re| re.captures(trimmed))
        {
            let id = captures[1].to_string();
            let model = captures[2].to_string();
            agents.push(CliAgent {
                name: format_agent_name(&id),
                id,
                source: source.clone(),
                default_model: if model == "inherit" { None } else { Some(model) },
            });
        }
    }

    Ok(agents)
}

fn format_agent_name(id: &str) -> String {
    // superpowers:code-reviewer → "代码审查 (superpowers:code-reviewer)"
    // pua:cto-p10 → "PUA:CTO (pua:cto-p10)"
    // Explore → "探索 (Explore)"
    // 简单实现：首字母大写
    id.split(':').last()
        .unwrap_or(id)
        .split('-')
        .map(|s| {
            let mut c = s.chars();
            match c.next() {
                None => String::new(),
                Some(f) => f.to_uppercase().collect::<String>() + c.as_str(),
            }
        })
        .collect::<Vec<_>>()
        .join(" ")
}
```

## 四、Phase 2：MCP 独立管理（3-4天）

### 4.1 MCP 服务器分类

```
MCP 服务器两种来源:
  ┌─────────────────────────────────────────────────┐
  │ 插件管理的 MCP (plugin:xxx:xxx)                  │
  │ 来源: 插件安装 → plugin.mcpServers               │
  │ 操作: 只能通过插件启用/禁用控制                    │
  │ 信息: type, url/command, args                     │
  │ 状态: 从 init 事件或 mcp list 获取                │
  ├─────────────────────────────────────────────────┤
  │ 用户手动添加的 MCP                                │
  │ 来源: claude mcp add / add-json                   │
  │ 操作: 可直接 add/remove/edit                      │
  │ 信息: type, scope, command, args, url, headers    │
  │ 状态: 从 mcp get 或 mcp list 获取                 │
  └─────────────────────────────────────────────────┘
```

## 八、风险与缓解

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| CLI 文本输出格式变更 | 中 | 解析失败 | 正则容错 + 降级到硬编码 |
| `claude agents` 命令不可用 | 低 | 无动态 Agent | PRESET_AGENTS 降级 |
| stream-json init 字段缺失 | 低 | 数据不完整 | 逐字段检查 null |
| MCP 认证需浏览器跳转 | 高 | 无法在 Tauri 内完成 | 打开系统浏览器 |
| `claude mcp list` 健康检查慢 | 高 | UI 阻塞 | 异步 + 超时 10s |
| 插件 MCP 无法通过 mcp 命令管理 | 确定 | 功能受限 | 区分来源，引导到插件管理 |
