# ⚙️ 三、Workflow 数据模型

## Workflow

```ts
Workflow {
  id
  name
  status (idle/running/paused)
  mode (continuous | scheduled | event)
  templateId
  currentNodeId
  memoryRoot
}
```

## Node

```ts
WorkflowNode {
  id
  workflowId
  role (product/dev/test/research)
  templateId
  triggerType (start | event | dependency)
  nextNodes[]
}
```

## Execution Record

```ts
Execution {
  id
  nodeId
  round
  status
  startTime
  endTime
  summaryPath
}
```

# 🔁 四、连续执行引擎（改造点2）

核心思想：

```
while(workflow.running){

   node = pickNextRunnableNode()

   runAgent(node)

   if(next exists):
       continue

}
```

特点：

* 不等待 cron
* 完成即进入下一轮
* 支持 pipeline 推进

# 🧠 六、Memory 系统（重点）

## Memory 分层结构

```
memory/

 active/
 summaries/
 archives/
 checkpoints/
 semantic/
 tasks/
 user_inputs/
```

---

## Memory 生命周期

### 触发压缩条件

* > 1500 行
* > 60k token
* 完成节点 >=5
* 阶段切换
* workflow idle

---

## 压缩流程

```
读取 active
抽取：
  已完成目标
  决策
  TODO
  风险
生成 summary
移动旧 → archives
写 semantic index
写 checkpoint
替换 active
```

---

## Summary 格式（强制结构化）

```
## Completed
## Decisions
## Current Focus
## Pending
## Risks
```

---

## Progressive Summarize

```
round → phase → milestone → project
```
