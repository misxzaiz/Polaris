# 三、核心数据模型

## 1️⃣ Workflow（任务流）

```ts
interface Workflow {

  id: string

  name: string

  templateId: string

  status:
    | 'CREATED'
    | 'PLANNING'
    | 'RUNNING'
    | 'WAITING_EVENT'
    | 'BLOCKED'
    | 'FAILED'
    | 'COMPLETED'
    | 'EVOLVING'

  priority: number

  continuousMode: boolean

  createdAt: number

}
````

---

## 2️⃣ Workflow Node（Agent节点）

```ts
interface WorkflowNode {

  id: string

  workflowId: string

  agentProfileId: string

  state:
    | 'IDLE'
    | 'READY'
    | 'RUNNING'
    | 'WAITING_INPUT'
    | 'DONE'
    | 'FAILED'

  subscribeEvents: string[]

  emitEvents: string[]

  maxRounds: number

  currentRounds: number

}
```

---

## 3️⃣ AgentProfile（专家模板）

```ts
interface AgentProfile {

  id: string

  role: string

  systemPolicy: string

  executionStrategy:

    | 'PLAN_FIRST'
    | 'CODE_FIRST'
    | 'TEST_DRIVEN'
    | 'EXPLORE'

  scoringRule: string

  selfEvolve: boolean

}
```

---

## 4️⃣ Event

```ts
interface AgentEvent {

  id: string

  type: string

  payload: any

  workflowId: string

  createdAt: number

  consumed: boolean

}
```

# 四、执行流程（Event Driven）

## ⭐ Orchestrator Loop

```
loop:

  1. poll event queue

  2. find nodes READY

  3. push to priority dispatcher

  4. spawn agent execution

  5. on finish → emit events

  6. update workflow lifecycle

  7. if continuousMode → requeue
```

---

# 五、Continuous Mode

```
node DONE

if workflow.continuousMode:

   node.state = READY
```

无需等待 cron。
