# 🧠 一、现在的 Scheduler 本质结构（真实定位）

从文档看：

### ✔ 当前是

> **单节点轮询式任务执行器**

核心机制：

```
调度循环（10秒）
   ↓
找 nextRunAt 到期任务
   ↓
build prompt
   ↓
spawn AI session
   ↓
执行完成
   ↓
计算 nextRunAt
```

### ✔ 协议模式 = 文件驱动的「任务状态机」

其实就是：

```
task.md = agent system prompt
memory/*.md = agent state
user-supplement.md = external interrupt queue
```

这个设计是非常对的 👍
但还差一个：

> ❗ **没有任务流（Workflow Graph）**

所以：

* 无法生产者消费者
* 无法 pipeline
* 无法协同 agent
* 无法并发 orchestration
* 无法即时触发

## ✅ 改造点2：执行完成立即下一轮

现在：

```
必须等 nextRunAt
```

这是 **Scheduler 思维**

而 AgentOS 应该是：

```
task finished
    ↓
requeue immediately
```

即：

### ⭐ 引入

```
continuous_mode: true
```

调度器不再看时间：

而是：

```
任务执行完 → push back queue
```

调度循环变成：

```
Event Driven Loop
```

不是 timer loop。
